Arrays and Hashes

January 11, 2015

If you're ever finding yourself programming and you have a large set of data there's two different ways to dealing with it: arrays and hashes. Arrays are ordered and indexed using integers. The items in an array can be called using their index number. These index numbers start at 0 and go up from there... So:

my_array[0] calls the first item in the array.
my_array[5[ calls the sixth item in the array.

Hashes work a bit differently. They're inherently unordered. They are indexed using key value pairs. These keys are object that correspond to specific values that are stored within the hash. The items in the hash can be called using their key, similar to how one calls using an array.

my_hash["dog"]
my_hash["salsa"]

Both definitely have their place within the ruby programming language. Hashes are great when the order is not the most important aspect or if ordering them does not make sense. Arrays are great for ordered things or if the order matters.
